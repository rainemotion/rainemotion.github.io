最速下降法

一. 最速下降法 (Steepest Descent Method) 原理

​ 最速下降法是一种经典的优化算法，用于寻找函数的局部最小值。它的核心思想是：在每一步迭代中，沿着当前位置函数梯度方向的反方向（即函数值下降最快的方向）移动，直到找到一个局部最小值。

1. 目标与基本思想

  • 目标： 给定一个目标函数 f(x)f(\mathbf{x})f(x)（其中 x\mathbf{x}x 可以是标量或向量），最速下降法的目标是找到一个 x\mathbf{x}x 值，使得 f(x)f(\mathbf{x})f(x) 达到其局部最小值。
  • 基本思想： 函数的梯度 ∇f(x)\nabla f(\mathbf{x})∇f(x) 指向函数值增长最快的方向。因此，沿着梯度的负方向 −∇f(x)-\nabla f(\mathbf{x})−∇f(x) 移动，可以确保函数值下降最快。

2. 迭代公式

最速下降法的迭代过程可以表示为：
xk+1=xk−αk∇f(xk)\mathbf{x}_{k+1} = \mathbf{x}_k - \alpha_k \nabla f(\mathbf{x}_k)xk+1​=xk​−αk​∇f(xk​)
其中：

  • xk\mathbf{x}_kxk​ 是第 kkk 次迭代时的当前点。
  • xk+1\mathbf{x}_{k+1}xk+1​ 是第 k+1k+1k+1 次迭代后的新点。
  • ∇f(xk)\nabla f(\mathbf{x}_k)∇f(xk​) 是函数 fff 在点 xk\mathbf{x}_kxk​ 处的梯度向量。它指明了函数在该点增长最快的方向和速率。
  • −∇f(xk)-\nabla f(\mathbf{x}_k)−∇f(xk​) 是下降方向，也称为最速下降方向。
  • αk\alpha_kαk​ 是步长（或学习率），它决定了在下降方向上移动的距离。选择合适的步长是算法成功的关键。

3. 算法步骤

  1. 初始化： 选择一个初始点 x0\mathbf{x}_0x0​ 和一个停止准则（例如，最大迭代次数，或梯度范数小于某个阈值）。
  2. 计算梯度： 在当前点 xk\mathbf{x}_kxk​ 处计算函数的梯度 ∇f(xk)\nabla f(\mathbf{x}_k)∇f(xk​)。
  3. 确定步长 αk\alpha_kαk​： 这是最速下降法中一个非常重要的步骤。步长可以是固定值，也可以通过线搜索（如 Armijo 线搜索或 Wolfe 线搜索）动态确定。
  4. 更新点： 根据迭代公式 xk+1=xk−αk∇f(xk)\mathbf{x}_{k+1} = \mathbf{x}_k - \alpha_k \nabla f(\mathbf{x}_k)xk+1​=xk​−αk​∇f(xk​) 更新到新的点。
  5. 检查停止准则： 如果满足停止准则，则算法终止；否则，返回步骤 2 继续迭代。

4. 步长的选择（线搜索）

步长 αk\alpha_kαk​ 的选择对最速下降法的收敛速度和稳定性至关重要：

  • 如果 αk\alpha_kαk​ 太小，收敛速度会非常慢，需要很多次迭代才能接近最小值。
  • 如果 αk\alpha_kαk​ 太大，可能会导致算法在最小值附近震荡，甚至发散，无法收敛。
  • 理想的 αk\alpha_kαk​ 应该使得 f(xk−αk∇f(xk))f(\mathbf{x}_k - \alpha_k \nabla f(\mathbf{x}_k))f(xk​−αk​∇f(xk​)) 最小化，但这通常是一个一维优化问题，称为精确线搜索，计算成本较高。

为了在计算效率和收敛性之间取得平衡，通常使用 非精确线搜索 方法来确定步长，如 Armijo 线搜索和 Wolfe 线搜索。

二. 线搜索方法原理

线搜索方法的目标是找到一个合适的步长 αk\alpha_kαk​，使得函数值在下降方向上有所减少，并且这种减少是“足够”的。

1. 固定步长 (Fixed Step Size)

  • 原理： 每次迭代都使用预先设定的一个常数作为步长 α\alphaα。

  • 示例 : alpha=0.1;

  • 优点： 实现简单，无需额外计算。

  • 缺点：

    • 不适应函数曲率的变化：对于不同的函数或优化过程的不同阶段，一个固定的步长可能过大或过小。
    • 收敛慢：如果步长太小。
    • 可能发散或震荡：如果步长太大。

2. Armijo 线搜索 (Armijo Line Search)

  • 原理： Armijo 线搜索是一种非精确线搜索方法，它基于“足够的下降”条件来确定步长。它要求新的函数值与当前函数值相比，至少有一个预设比例的下降量。

  • Armijo 条件：
    f(x−αg)≤f(x)−cα∥g∥2f(\mathbf{x} - \alpha \mathbf{g}) \le f(\mathbf{x}) - c \alpha \|\mathbf{g}\|^2f(x−αg)≤f(x)−cα∥g∥2
    其中：

    • x\mathbf{x}x 是当前点。
    • g\mathbf{g}g 是在 x\mathbf{x}x 处的梯度 ∇f(x)\nabla f(\mathbf{x})∇f(x)。
    • α\alphaα 是试探步长。
    • ccc 是一个小的正数，通常在 0<c<10 < c < 10<c<1 之间（文档中示例为 1e−41e-41e−4）。它控制了“足够的下降”的严格程度。
    • ∥g∥2\|\mathbf{g}\|^2∥g∥2 是梯度的 L2 范数的平方，即 gTg\mathbf{g}^T \mathbf{g}gTg。

  • 工作机制 (文档中的 armijo_line_search 函数):

    1. 从一个初始的试探步长 α\alphaα 开始（通常是 1）。

    2. 检查 Armijo 条件。

    3. 如果条件不满足

      f(x−alpha∗grad)>f(x)−c∗alpha∗norm(grad)2f(x-alpha *grad) > f(x)-c* alpha*norm(grad)^2 f(x−alpha∗grad)>f(x)−c∗alpha∗norm(grad)2

      ，说明当前步长 α\alphaα 太大，下降不够“足够”，则将 α\alphaα 减半（alpha = alpha / 2;）。

    4. 重复步骤 2 和 3，直到找到一个满足 Armijo 条件的 α\alphaα。

  • 优点：

    • 实现相对简单。
    • 保证每次迭代函数值都有“足够的下降”。
    • 相对于精确线搜索，计算成本较低。

  • 缺点：

    • 只保证了足够的下降，但可能没有充分利用下降方向的信息，有时会选择过小的步长，导致收敛速度变慢。
    • 可能导致步长在迭代过程中迅速减小，甚至接近 0。

3. Wolfe 线搜索 (Wolfe Line Search)

  • 原理： Wolfe 线搜索是比 Armijo 线搜索更严格的线搜索方法，它包含两个条件：Armijo 条件（足够的下降）和曲率条件（保证步长不会太小，并且梯度有足够的减小）。

  • Wolfe 条件：

    1. Armijo 条件 (足够的下降条件):
      f(x−αg)≤f(x)−c1αgTgf(\mathbf{x} - \alpha \mathbf{g}) \le f(\mathbf{x}) - c_1 \alpha \mathbf{g}^T \mathbf{g}f(x−αg)≤f(x)−c1​αgTg
      这与 Armijo 线搜索中的条件类似，确保函数值有足够下降。c1c_1c1​ 通常是一个较小的正数（如 0.1），且 0<c1<10 < c_1 < 10<c1​<1。

    2. 曲率条件 (Curvature Condition):
      ∇f(x−αg)Tg≥c2∇f(x)Tg\nabla f(\mathbf{x} - \alpha \mathbf{g})^T \mathbf{g} \ge c_2 \nabla f(\mathbf{x})^T \mathbf{g}∇f(x−αg)Tg≥c2​∇f(x)Tg
      或者对于强 Wolfe 条件，通常写作 ∣∇f(x−αg)Tg∣≤c2∣∇f(x)Tg∣\vert\nabla f(\mathbf{x} - \alpha \mathbf{g})^T \mathbf{g}\vert \le c_2 \vert\nabla f(\mathbf{x})^T \mathbf{g}\vert∣∇f(x−αg)Tg∣≤c2​∣∇f(x)Tg∣。文档中给出的形式是标准 Wolfe 条件。

      • 这个条件确保步长 α\alphaα 不会太小。它要求在新的点 x−αg\mathbf{x} - \alpha \mathbf{g}x−αg 处的梯度 ∇f(x−αg)\nabla f(\mathbf{x} - \alpha \mathbf{g})∇f(x−αg) 沿着下降方向 g\mathbf{g}g 的投影，必须大于或等于原始梯度投影的一个比例。
      • 直观地说，它要求新的梯度与下降方向的“夹角”不能太大，即梯度在下降方向上的分量不能变得太小（这意味着我们已经“越过”了最小值并且梯度方向开始反转）。
      • c2c_2c2​ 通常是一个比 c1c_1c1​ 大的数（文档中示例为 0.9），且 0<c1<c2<10 < c_1 < c_2 < 10<c1​<c2​<1。

  • 工作机制 ( wolfe_line_search 函数):

    1. 从一个初始的试探步长 α\alphaα 开始（通常是 1）。
    2. 计算新的点 new_x = x - alpha * grad;
    3. 检查 Armijo 条件： 如果 f(new_x) > f(x) - c1 * alpha * grad' * grad，说明下降不够充分，步长 α\alphaα 太大，减半 (alpha = alpha / 2;)。
    4. 检查曲率条件： 如果 Armijo 条件满足，接着检查 grad_f(new_x)' * grad >= c2 * grad_f(x)' * grad。
      • 如果满足，说明找到了一个合适的步长，退出循环 (break;)。
      • 如果不满足，说明步长可能太小，或者还需要调整，文档中的实现是继续减半 (alpha = alpha / 2;)。一个更完善的 Wolfe 线搜索实现通常会涉及二分法或插值来更有效地找到满足条件的区间。

  • 优点：

    • 比 Armijo 线搜索更鲁棒，能够找到一个相对更“最优”的步长。
    • 保证了函数值有足够的下降，并且步长不会过小，有助于避免病态情况下的震荡。
    • 对于某些优化算法（如拟牛顿法），结合 Wolfe 线搜索可以保证超线性收敛。

  • 缺点：

    • 实现比 Armijo 线搜索更复杂。
    • 由于需要满足两个条件，每次迭代寻找步长可能需要更多的计算，导致单次迭代耗时更长。

4. 固定步长法的特点与理解

  • 优点： 实现简单，无需在每次迭代中进行复杂的计算来确定步长。

  • 缺点及关键点：

    • 步长 α\alphaα 的选择至关重要：

      • 如果 α\alphaα 太小（例如 0.0010.0010.001），算法会非常缓慢地收敛，需要大量的迭代次数才能接近最小值。每一步移动的距离太小，效率低下。
      • 如果 α\alphaα 太大（例如 111），可能会导致算法在最小值附近“跳过”或者震荡，甚至可能发散，无法收敛到最小值。对于这个函数 f(x)=(x−3)2f(x)=(x-3)^2f(x)=(x−3)2，其梯度是 2(x−3)2(x-3)2(x−3)。如果 α\alphaα 取 1，则 xk+1=xk−1×2(xk−3)=xk−2xk+6=−xk+6x_{k+1} = x_k - 1 \times 2(x_k-3) = x_k - 2x_k + 6 = -x_k + 6xk+1​=xk​−1×2(xk​−3)=xk​−2xk​+6=−xk​+6。如果 x0=0x_0=0x0​=0，则 x1=6x_1=6x1​=6，x2=0x_2=0x2​=0，x3=6x_3=6x3​=6，会在 0 和 6 之间来回震荡，永远无法收敛到 3。

    • 不适应函数曲率变化： 对于复杂函数，其曲率在不同区域可能差异很大。一个固定的步长可能无法很好地适应这种变化，导致在某些区域收敛慢，在另一些区域又可能过冲。

    • 适用于简单凸函数： 对于像本例这种简单的二次凸函数，如果 α\alphaα 选择得当，固定步长法可以有效地找到最小值。

四. Armijo 线搜索

1. 解决的问题：寻找“足够好”的步长

在最速下降法中，每一次迭代都需要确定一个步长 αk\alpha_kαk​，用于更新当前点：
xk+1=xk−αk∇f(xk)\mathbf{x}_{k+1} = \mathbf{x}_k - \alpha_k \nabla f(\mathbf{x}_k)xk+1​=xk​−αk​∇f(xk​)
步长 αk\alpha_kαk​ 的选择至关重要：

  • 如果 αk\alpha_kαk​ 过小，算法收敛会非常缓慢。
  • 如果 αk\alpha_kαk​ 过大，可能会导致函数值不下降、震荡，甚至发散。
  • 理想情况下，我们希望找到使 f(xk−αk∇f(xk))f(\mathbf{x}_k - \alpha_k \nabla f(\mathbf{x}_k))f(xk​−αk​∇f(xk​)) 最小的 αk\alpha_kαk​，但这通常是一个耗时的一维搜索问题（精确线搜索）。

Armijo 线搜索是一种 非精确线搜索 方法，它的目标不是找到最优步长，而是高效地找到一个“足够好”的步长，保证函数值有**“足够的下降”**，从而使算法能够稳定且合理地收敛。

2. 核心条件：Armijo 条件 (Sufficient Decrease Condition)

Armijo 线搜索的核心是一个数学不等式条件，用于判断一个试探步长 α\alphaα 是否满足“足够的下降”：
f(x−αg)≤f(x)−cα∥g∥2f(\mathbf{x} - \alpha \mathbf{g}) \le f(\mathbf{x}) - c \alpha \|\mathbf{g}\|^2f(x−αg)≤f(x)−cα∥g∥2
其中：

  • x\mathbf{x}x：当前迭代点。
  • g=∇f(x)\mathbf{g} = \nabla f(\mathbf{x})g=∇f(x)：函数 fff 在当前点 x\mathbf{x}x 处的梯度向量。
  • α\alphaα：当前正在试探的步长值（通常是正数）。
  • ccc: 一个预设的常数，称为 Armijo 参数或条件系数。它是一个很小的正数，通常在 0<c<10 < c < 10<c<1 之间（例如， 0.00010.00010.0001，或 10−410^{-4}10−4）。这个参数控制了我们期望的“足够的下降”的严格程度。
  • ∥g∥2\|\mathbf{g}\|^2∥g∥2: 梯度向量 g\mathbf{g}g 的 L2 范数（欧几里得范数）的平方，即 gTg\mathbf{g}^T \mathbf{g}gTg , 它表示梯度的大小。

3. 条件的直观理解

这个 Armijo 条件可以这样来理解：

  • 左侧 f(x−αg)f(\mathbf{x} - \alpha \mathbf{g})f(x−αg)：代表沿着梯度负方向（下降方向）移动 α\alphaα 步长后，新点处的实际函数值。

  • 右侧 f(x)−cα∥g∥2f(\mathbf{x}) - c \alpha \|\mathbf{g}\|^2f(x)−cα∥g∥2：

    • f(x)f(\mathbf{x})f(x) 是当前点的函数值。
    • α∥g∥2\alpha \|\mathbf{g}\|^2α∥g∥2 表示如果函数是完全线性的，那么沿着梯度方向下降 α\alphaα 步长，函数值理论上可以下降的“最大”量。
    • 由于实际函数通常不是线性的，我们不能期望每次都达到理想的最大下降。因此，我们引入一个折扣因子 ccc。
    • 所以，右侧可以看作是一个“安全下界”或“参考下降线”。它要求我们新点处的函数值必须低于当前函数值减去某个基准下降量。

简单来说，Armijo 条件要求实际的函数值下降量必须至少达到沿着线性近似下降量的某个比例 ccc。

  • 如果实际下降量太小（即左侧大于右侧），则说明步长 α\alphaα 太大，可能跳过了最小值，或者下降不够有效，需要减小步长。
  • 如果实际下降量足够（即左侧小于等于右侧），则认为步长 α\alphaα 是可接受的，并采纳它。

这个条件有效地防止了步长过大导致的函数值不下降或上升，也避免了步长过小导致下降过于缓慢的情况（因为如果 α\alphaα 过小，左侧也可能不满足条件）。

4. 算法流程：回溯线搜索 (Backtracking Line Search)

Armijo 线搜索通常采用一种回溯（Backtracking）策略来寻找合适的 α\alphaα：

  1. 初始化：

    • 选择一个初始的试探步长 αinitial\alpha_{initial}αinitial​ (通常设置为 1，因为我们希望步子尽量大)。
    • 选择 Armijo 参数 ccc (例如 0.00010.00010.0001)。
    • 选择一个步长缩减因子 ρ\rhoρ (通常为 0.5，即每次减半)。

  2. 迭代寻找 α\alphaα:
    在每次最速下降法的大迭代中，我们需要执行以下小循环来找到 αk\alpha_kαk​:

    • 从当前试探步长 α\alphaα 开始 (第一次通常是 αinitial\alpha_{initial}αinitial​)。

    • 检查 Armijo 条件：
      计算 f(x−αg)f(\mathbf{x} - \alpha \mathbf{g})f(x−αg) 和 f(x)−cα∥g∥2f(\mathbf{x}) - c \alpha \|\mathbf{g}\|^2f(x)−cα∥g∥2。

      • 如果条件满足

        (f(x−αg)≤f(x)−cα∣g∣2)(f(\mathbf{x} - \alpha \mathbf{g}) \le f(\mathbf{x}) - c \alpha |\mathbf{g}|^2) (f(x−αg)≤f(x)−cα∣g∣2)

      • 如果条件不满足

        (f(x−αg)>f(x)−cα∣g∣2)(f(\mathbf{x} - \alpha \mathbf{g}) > f(\mathbf{x}) - c \alpha |\mathbf{g}|^2) (f(x−αg)>f(x)−cα∣g∣2)

由于函数在下降方向是连续的，并且只要梯度非零，最终总能找到一个足够小的 α\alphaα 来满足 Armijo 条件。

五. Wolfe 线搜索

1. 解决的问题：找到“足够好”且“不过小”的步长

Armijo 线搜索虽然能够保证每次迭代函数值有“足够的下降”，但它有一个潜在的缺点：它可能会选择非常小的步长。这可能导致算法收敛缓慢，特别是在函数曲率变化剧烈或目标函数呈“长而窄的山谷”形状时，算法可能会在谷底两侧来回震荡，而无法高效地前进。

Wolfe 线搜索旨在解决这个问题，它在 Armijo 条件的基础上，额外引入了一个 曲率条件（Curvature Condition），确保步长不会太小，并且梯度在新的点处有足够的减小，从而使优化过程更加稳定和高效。

2. 核心条件：两个 Wolfe 条件

Wolfe 线搜索由两个条件组成：

(1) Armijo 条件 (足够的下降条件 / Sufficient Decrease Condition)
这个条件与 Armijo 线搜索的核心条件完全相同，确保新点处的函数值相对于当前点有显著的下降：
f(x−αg)≤f(x)−c1α∥g∥2f(\mathbf{x} - \alpha \mathbf{g}) \le f(\mathbf{x}) - c_1 \alpha \|\mathbf{g}\|^2f(x−αg)≤f(x)−c1​α∥g∥2
其中：

  • x\mathbf{x}x：当前迭代点。
  • g=∇f(x)\mathbf{g} = \nabla f(\mathbf{x})g=∇f(x)：函数 fff 在当前点 x\mathbf{x}x 处的梯度。
  • α\alphaα：当前试探的步长。
  • c1c_1c1​：Armijo 参数，一个很小的正数，通常在 0<c1<10 < c_1 < 10<c1​<1 之间（例如 0.10.10.1）。
  • ∥g∥2\|\mathbf{g}\|^2∥g∥2: 梯度的 L2 范数的平方，即 gTg\mathbf{g}^T \mathbf{g}gTg。

直观理解： 确保每一步都朝着正确的下降方向移动，并且下降量足够显著。

(2) 曲率条件 (Curvature Condition)
这个条件要求新点处的梯度 ∇f(x−αg)\nabla f(\mathbf{x} - \alpha \mathbf{g})∇f(x−αg) 在下降方向上的投影，要比当前点 ∇f(x)\nabla f(\mathbf{x})∇f(x) 的投影 更接近零（或者说，梯度的变化要足够大）。它防止步长过小：
∇f(x−αg)Tg≥c2∇f(x)Tg\nabla f(\mathbf{x} - \alpha \mathbf{g})^T \mathbf{g} \ge c_2 \nabla f(\mathbf{x})^T \mathbf{g}∇f(x−αg)Tg≥c2​∇f(x)Tg
其中：

  • ∇f(x−αg)\nabla f(\mathbf{x} - \alpha \mathbf{g})∇f(x−αg)：新点处的梯度。
  • g\mathbf{g}g：当前点处的梯度。
  • c2c_2c2​：曲率参数，一个常数，通常在 c1<c2<1c_1 < c_2 < 1c1​<c2​<1 之间（例如 0.9）。

直观理解：

  • ∇f(x)Tg\nabla f(\mathbf{x})^T \mathbf{g}∇f(x)Tg (即 ∥g∥2\|\mathbf{g}\|^2∥g∥2) 代表当前点梯度在梯度方向上的投影（总是负的）。
  • ∇f(x−αg)Tg\nabla f(\mathbf{x} - \alpha \mathbf{g})^T \mathbf{g}∇f(x−αg)Tg 代表新点梯度在 原下降方向 上的投影。
  • 这个条件要求新点处的“斜率”（在原下降方向上）必须比当前点的“斜率”更平坦（即更小或更接近零）。它确保我们没有选择一个过小的步长，以至于还在函数值的陡峭区域（梯度方向变化不大）。如果步长太小，函数曲面几乎是线性的，梯度方向几乎不变，这个条件就不会满足。

强 Wolfe 条件 (Strong Wolfe Conditions):
在实际中，有时会使用更严格的 强 Wolfe 条件，它在曲率条件上增加了一个绝对值，要求新点处梯度在搜索方向上的投影的绝对值要小于当前梯度投影的绝对值：
∣∇f(x−αg)Tg∣≤c2∣∇f(x)Tg∣\vert \nabla f(\mathbf{x} - \alpha \mathbf{g})^T \mathbf{g} \vert \le c_2 \vert \nabla f(\mathbf{x})^T \mathbf{g} \vert∣∇f(x−αg)Tg∣≤c2​∣∇f(x)Tg∣
这个强形式可以更有效地防止步长过大导致的“过冲”（overshooting），即步子迈得太大，以至于跳过最小值点太多，导致梯度方向反转过大。我们这里代码使用的是 弱 Wolfe 条件 的形式。

3. 算法流程 (如何寻找步长 α\alphaα)

Wolfe 线搜索寻找 α\alphaα 的过程通常比 Armijo 更复杂，因为它需要同时满足两个条件。它通常也采用回溯（Backtracking）或插值/二分法相结合的策略：

  1. 初始化： 选择初始试探步长 αinitial\alpha_{initial}αinitial​ (通常是 1)，以及参数 c1c_1c1​ 和 c2c_2c2​。

  2. 迭代寻找 α\alphaα:

    • 从当前试探步长 α\alphaα 开始。

    • 首先检查 Armijo 条件：

      • 如果 Armijo 条件不满足（函数下降不够），说明 α\alphaα 太大。减小 α\alphaα (例如 α←ρα\alpha \leftarrow \rho \alphaα←ρα)，然后重新检查。

    • 如果 Armijo 条件满足，接着检查曲率条件：

      • 如果曲率条件不满足（通常表示 α\alphaα 可能仍然太小，或者不足以使梯度充分改变），通常会增大 α\alphaα（通过插值或二分法在一个区间内搜索），或者根据文档中的实现，有时也会选择减小 α\alphaα 继续尝试。

  • 双重保障： 结合 Armijo 条件（足够的下降）和曲率条件（防止步长过小），使得步长选择更加稳健。
  • 防止震荡： 曲率条件有效地避免了在狭窄山谷中步长过小导致的慢收敛或震荡问题。
  • 更强的收敛性： 与一些更高级的优化算法（如拟牛顿法）结合时，Wolfe 条件可以保证超线性收敛。
  • 计算成本更高： 相比 Armijo 线搜索，每次线搜索迭代需要额外计算新点的梯度，并检查第二个条件，因此单次线搜索的计算成本更高。

1. 定义目标函数及其梯度

  • f = @(x) x.^4 - 3*x.^3 + 2;

  • x_armijo = x0;, path_armijo = x_armijo;, f_values_armijo = f(x0);

    • 开始 Armijo 方法的迭代循环。

  • grad = grad_f(x_armijo);

    • 计算当前点 x_armijo 处的梯度。

  • if abs(grad) < 1e-6 ... break; end

    • 这是一个提前停止条件。如果梯度的绝对值小于 1e-6（一个很小的阈值），则认为已经足够接近局部最优，停止迭代。这比单纯依赖 max_iter 更智能。

    • 根据找到的步长更新 x_armijo。

七. Armijo 与 Wolfe 线搜索的速度差异

  • Armijo 线搜索通常更快的原因：

    • 条件较弱： 它只关注“足够的下降”，即只要求满足一个条件。
    • 容易满足： 由于条件宽松，步长更容易满足条件并被接受。这通常导致步长更新过程更快，迭代速度看起来较快。

  • Wolfe 线搜索可能较慢的原因：

    • 条件更严格： 它包括 Armijo 条件和曲率条件两个。
    • 更谨慎的步长更新： 步长必须同时满足这两个条件，这使得步长更新过程更为谨慎，需要更多尝试才能找到合适的 α\alphaα。
    • 避免过大步长导致震荡： 曲率条件能够更好地控制步长，避免过大的步长导致震荡或错过最优解。

总而言之，Armijo线搜索追求快速找到一个“可行”的步长，而Wolfe线搜索则追求找到一个更“优化”的步长，它在保证下降的同时，还对步长的“质量”提出了要求，从而使得整个优化过程更稳定和高效（尽管单次迭代可能耗时更长）。
